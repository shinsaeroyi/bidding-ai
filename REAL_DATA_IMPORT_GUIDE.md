# 실제 나라장터 데이터로 교체하는 방법

현재 `sample_data.csv`와 앱 시작 시 자동 생성되는 데이터는 실제 사례가 아니라 MVP 기능 확인용 가상 데이터입니다.

## 필요한 공개 데이터

공공데이터포털에서 아래 서비스를 활용신청합니다.

1. 조달청_나라장터 낙찰정보서비스
   - 개찰결과, 최종낙찰자, 개찰순위, 복수예비가 및 예비가격 정보
   - 서비스 페이지: https://www.data.go.kr/data/15129397/openapi.do

2. 조달청_나라장터 입찰공고정보서비스
   - 공고 목록, 공고 상세, 기초금액 정보, 참가가능지역, 면허제한 정보
   - 서비스 페이지: https://www.data.go.kr/data/15129394/openapi.do

3. 전국낙찰정보표준데이터 또는 나라장터 공공데이터개방표준서비스
   - 표준화된 입찰/낙찰/계약 정보 확인용
   - 서비스 페이지: https://www.data.go.kr/data/15023681/standard.do

## MVP에 필요한 최소 컬럼

CSV 업로드 시 최소한 아래 컬럼이 필요합니다.

- 공고번호
- 공고명
- 기초금액
- 예정가격
- 낙찰하한율

앱은 아래 컬럼을 자동 계산하거나 선택적으로 사용합니다.

- 사정률 = 예정가격 / 기초금액 * 100
- 낙찰하한가 = 예정가격 * 낙찰하한율 / 100
- 발주기관
- 지역
- 용역구분
- 개찰일
- 입찰참가자 수
- 낙찰금액
- 투찰률
- 복수예비가격 범위

## 실제 데이터 구성 시 주의

- 감리용역, 건설사업관리용역, 시공단계 건설사업관리용역만 필터링합니다.
- `±2%` 공고와 `±3%` 공고는 섞지 않습니다.
- 기초금액, 예정가격, 낙찰하한율이 없는 행은 모델 학습에서 제외합니다.
- 낙찰하한율은 `87.745`처럼 퍼센트 단위로 저장합니다.
- 예정가격이 0이거나 기초금액 대비 비정상적인 사정률인 행은 검수합니다.

## 다음 구현 단계

API 키를 받으면 다음 기능을 추가할 수 있습니다.

1. 앱 `Data Upload` 탭에 공공데이터포털 서비스키 입력칸 추가
2. 개찰일 기간, 업무구분, 검색어 입력
3. 낙찰정보서비스에서 개찰결과 조회
4. 입찰공고정보서비스에서 기초금액 조회
5. 공고번호 기준으로 병합
6. `sample_data.csv` 대신 실제 CSV 저장
7. 앱 세션 데이터를 실제 데이터로 교체
